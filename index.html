Making 1

Making 2

For this website I was really inspired to create some sort of "game." I decided that tennis would be the game. Admittedly, a lot of this website was programmed by AI but I did understand how some of the coding was made.

Making 3

I learned from my last website that I wouldn't get very far if I kept on depending on AI to do the code for me because I wouldn't ever understand what the AI was coding for me. For this multipage website I was very proud with my own coding and ability to work through my obstacles by being a "pro hacker." I successfully made a website all about my favorite things with working buttons to take the user to information slides.

Phase 1 Course Goals


1. Foster the creative habits of mind

I learned how to use code to build websites that match my own ideas. At first, I only followed the examples, but now I can adjust the code to fit what I want. This process showed me that coding is a creative tool, not just rules to follow. When I test changes and see them work, it gives me confidence to try new ideas.

2. Engage with others in a workshop environment.

Working with classmates helped me see different ways to solve the same coding problem. Sometimes I missed an easy fix, and someone else showed me. Other times, I explained my solution to them. Talking about code made me understand it better. The workshop setting made learning less stressful because I knew I was not working alone.

3. Think deeply about craft.

I started to notice how small details matter in code. Even a missing semicolon can break a whole page. I also realized design choices, like where to place buttons or how to link pages, affect how people use a site. Thinking about these details changed how I write and check my work. It made me focus not just on getting it to run, but on making it work well.

4. Engage in sustained research and skills development.

Coding a website took practice and patience. I often had to look up how certain tags or commands worked. At first, it felt slow, but I learned to read instructions and apply them to my own code. I saw progress each week as I tried new things. Keeping at it showed me that skills grow with steady practice, not all at once.

5. Learn to present your work

When I shared my website with others, I had to explain what I did and why. This helped me see my project from the user’s view, not just the coder’s. Presenting forced me to keep things simple and clear. I learned to show the main parts first, then explain the details if someone asked. It made me more confident in showing my work.
